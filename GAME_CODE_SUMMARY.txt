================================================================================
BALL PIT X  —  CODEBASE SUMMARY  (for future sessions)
================================================================================
Last updated: 2026-06 (after powerup-physics + multi-bounce level-orb work)

A LÖVE2D (Love2D 11.3, Lua) roguelike that fuses Arkanoid/Breakout with a
vampire-survivors swarm loop and an ULTRAKILL-style combo meter. You control a
paddle; your "heroes" are bouncing balls (SNKRX archetypes) that auto-attack;
enemy "bricks" drift down in swarms and damage you if they reach the paddle.

--------------------------------------------------------------------------------
0. CRITICAL THINGS TO KNOW FIRST
--------------------------------------------------------------------------------
* DUAL-BUILD SYNC. There are TWO parallel copies of the game:
    - BlockBreakoutGame/   (uses LF line endings)
    - admin/               (uses CRLF line endings; this is the PLAYTEST build,
                            it additionally has admin/terminal.lua, a debug
                            console that can spawn powerups / force state)
  RULE: every GAMEPLAY change must be applied to BOTH copies.
  - For most files the two copies are byte-identical (modulo line endings), so
    the same Edit old_string/new_string works in both. Verify with:
        diff <(tr -d '\r' < admin/FILE) <(tr -d '\r' < BlockBreakoutGame/FILE)
  - EXCEPTION: ballpit.lua DIFFERS between builds. admin/ballpit.lua has
    terminal hooks and a multi-line if/elseif in apply_powerup (with an
    "unknown powerup" terminal log) plus extra comments. Function bodies are
    equivalent but NOT byte-identical, and line numbers differ
    (admin is ~+350 lines offset). Find anchors per-file before editing.

* CANNOT RUN THE GAME HERE. No Lua interpreter in this environment and LÖVE
  opens a GUI window. All gameplay/UI edits are UNTESTED until the user runs
  BlockBreakoutGame\run.bat (LÖVE launcher). Always say edits are untested.

* LINE NUMBERS SHIFT. The user edits files concurrently (and a linter touches
  them). Re-grep for anchors; don't trust cached line numbers.

* COORDINATE SYSTEM. Love2D is y-DOWN. graphics.push(x,y,r) rotates around a
  point. graphics.triangle points RIGHT at angle 0; push by -math.pi/2 to make
  it point UP. Canvas is fixed gw=480 x gh=656, scaled up to the window.

--------------------------------------------------------------------------------
1. ENGINE  (BlockBreakoutGame/engine/ — a327ex/SNKRX-style framework)
--------------------------------------------------------------------------------
Third-party-ish framework; you rarely edit it. Loaded via `require 'engine'`.
Key facilities used by the game:

* Object        : class system. `Foo = Object:extend()`, `Foo:implement(Mixin)`.
* GameObject    : base mixin. init_game_object/update_game_object; gives every
                  entity x,y, an id, a self.t (Trigger), self.spring, self.hfx
                  (hit-flash fx), group membership, self.dead flag.
* Physics       : Box2D wrapper mixin (engine/game/physics.lua). Methods:
                  set_as_circle(r,type,tag) / set_as_rectangle(w,h,type,tag),
                  set_restitution/friction/damping/mass/fixed_rotation,
                  set_velocity/get_velocity, set_angle/get_angle,
                  set_angular_velocity/set_angular_damping/apply_angular_impulse,
                  apply_force, set_position, update_position (syncs x,y, NOT
                  angle). Collision via self.on_collision_enter = fn(self,other,
                  contact) and other.tag checks.
* State         : screen/state mixin. BallPit implements it (on_enter, etc.).
* Group         : entity container + optional Box2D world. Created in BallPit:
                  reset_run as self.main (physics), self.swarms, self.effects,
                  self.ui. set_as_physics_world(meter, gx, gy, {collision_tags}),
                  disable_collision_between(tagA, tagB), no_camera().
* Trigger (self.t): per-object timers. after(delay,fn,tag) / every / cooldown /
                  during / tween / run; reset(tag) restarts; cancel(tag) stops.
                  Re-calling after() with the same tag overwrites (safe refresh).
                  NOTE: a one-shot `after` deletes its tag when it fires, so use
                  re-armed after() rather than reset() if it may have expired.
* input         : input:bind('action', {keys}); read input.action.down/pressed.
* Sound/SoundTag: ripple-based audio (engine/external/ripple.lua). sfx & music
                  tags. engine/sound.lua is a stub ("TODO implement later").
* graphics      : drawing module. rectangle(x,y,w,h,rx,ry,color,line_width),
                  circle(x,y,r,color,line_width), triangle(x,y,w,h,color,lw),
                  push(x,y,r[,sx,sy])/pop, print/print_centered, set_color, etc.
                  (line_width arg = draw as outline.)
* Color         : Color(r,g,b,a) or Color'#hexstr'; :clone(), :lighten(amt).
* Font, Canvas, Shader, Camera, Grid, random (random:int/float/bool/uid), math
  helpers (math.clamp/remap/distance/atan2/linear...), polygon utils.

Engine subfolders: datastructures/ (string, grid, binser), external/ (ripple
audio, clipper, binser serialization), graphics/ (graphics, camera), game/
(physics, trigger, ...), math/ (math, random, polygon), sound.

2. BOOT / ENTRY FLOW
--------------------------------------------------------------------------------
conf.lua        : love.conf — version 11.3, identity "BallPitX", window 720x984,
                  vsync on, title "Ball Pit X".

main.lua        : Entry point.
                  - requires all game modules (order: engine, shared, ballpit,
                    paddle, ball_hero, brick, swarm, enemies, projectile,
                    xp_orb, powerup, effects).
                  - init(): shared_init(); binds input (WASD=move, arrows=aim,
                    space=launch, enter=confirm, m1=click, r=restart); loads
                    ~30 SFX + 5 music tracks (Kubbi - Ember album); creates
                    main = Main() and adds BallPit'ballpit', go_to('ballpit').
                  - update(dt)=main:update; draw()=shared_draw(main:draw).
                  - love.run = engine_run{game 480x656, window 720x984, vsync}.

shared.lua      : Global palette, fonts, canvases, hero tables.
                  - ColorRamp: a Color plus lighter/darker variants indexed
                    -10..10 ([0]=base, [-2]=darker, [5]=lighter).
                  - shared_init(): builds palette globals (white, black, bg, fg,
                    fg_alt, yellow, orange, blue, green, red, purple, blue2,
                    yellow2) + _transparent / _transparent_weak variants; fonts
                    fat_font & pixul_font (8px); canvases background_canvas,
                    main_canvas (stencil), shadow_canvas + shadow_shader;
                    character_colors (per-hero tint, ~58 heroes); hero_pool
                    (the full 58-hero draft list).
                  - shared_draw(): renders a checkerboard bg grid to
                    background_canvas, the game to main_canvas, a drop shadow to
                    shadow_canvas, then composites all three.

--------------------------------------------------------------------------------
3. BALLPIT.LUA — the orchestrator (one big State; ~2400+ lines)
--------------------------------------------------------------------------------
Contains the Wall class + the BallPit State. Organized by subsystem:

[Walls]        Wall: static rectangle, restitution 1, friction 0 (FRICTIONLESS —
               important: bounces preserve spin/speed cleanly). Invisible.

[Module consts] RANGED_VARIANTS + RANGED_SPAWN_SCALE (=0.5; thins ranged enemy
               spawn weights so bullets don't blanket the screen).
               COMBO_RANKS (D/C/B/A/S/SS/SSS/FRENZY with thresholds 0/50/150/
               300/500/750/1100/1500 and mults 1.0..4.0) and COMBO_* tunables
               (penalty_miss 150, base_points 10, variety_bonus 5,
               streak_bonus_cap 10, idle_grace 2s, idle_decay 20/s,
               bounce_dmg_step 0.08, bounce_cap 15).

[wave_config(wave)]  Per-wave tuning. Returns enemy `mix` ({variant,weight}
               pairs), swarm_interval, duration, rows min/max, width fractions,
               drift_speed (scaled by live arena height), min_swarm_gap.
               Wave 10 = boss wave (disables swarm spawner). Difficulty slides
               up with wave; waves 11+ are the hardest post-boss tier.
               append_ranged() helper folds ranged variants into late mixes.

[Window/Settings]  scale_options (0.75x..1.25x of 480x656), apply_scale_option,
               settings menu UI (update_settings/draw_settings), a settings
               screen that also shows the hero roster
               (hero_grid_pos/draw_settings_heroes).

[Tutorial]     open_tutorial, tutorial pages, draw_tutorial + draw_tutorial_
               diagram (animated how-to-play diagrams), tut_arrow helper.

[on_enter / reset_run]  reset_run REBUILDS the whole run:
               - Creates physics world self.main with collision tags
                 {paddle, ball, brick, wall, xp, projectile, powerup}.
               - COLLISION MATRIX (see section 9).
               - Arena bounds x1,y1=24,18  x2,y2=gw-24,gh-24. Spawns LEFT,
                 RIGHT, TOP walls; BOTTOM IS OPEN (missed balls fall into the
                 pit and magnet-recall to the paddle). self.top_wall captured so
                 ball pierce expires on ceiling bonk.
               - Paddle at bottom; starting heroes = vagrant + swordsman.
               - Run state: player_hp 5/5, xp/level/xp_to_next, wave 1, score,
                 combo table, stuck-ball aim state, buffs table, powerup_pity
                 (chance/streak model) and levelup_pity (timed model).

[Wave/Swarm]   start_wave, spawn_boss (wave 10), advance_wave (next wave +
               guaranteed tier-2 powerup drop), roll_variant (weighted pick from
               mix). Swarm PLACEMENT: arena_center_x, snap_to_grid_x,
               expand_to_cells, can_place_layout (overlap test vs live grid),
               zone_occupancy, pick_swarm_anchor, spawn_swarm.

[update/draw]  update(dt): drives waves, timers (combo/pity/buffs), boss gating
               (awaiting_boss holds wave 10 until arena cleared), input. draw():
               world + HUD + aim line + overlays (settings/tutorial/upgrade/
               game over). draw_hud, draw_aim_line.

[Combo system] on_brick_killed, combo_rank_index, combo_mult, bounce_dmg_mult,
               on_brick_bounce (awards points: base + variety + streak bonus;
               scales per-ball bounce damage), on_combo_rank_up, on_ball_missed
               (penalty + streak wipe), tick_combo (idle decay), draw_combo_meter.

[Breach/Damage] on_brick_breached / on_row_breached (enemy reached paddle ->
               player takes damage), heal_player, trigger_game_over,
               draw_game_over. Death at player_hp <= 0.

[XP/Level/Draft] gain_xp (can cross multiple level thresholds -> queues
               pending_levelups), level_up, offer_upgrades (builds a 3-card
               draft), update_upgrade (mouse/keys), upgrade_card_under_mouse,
               confirm_upgrade (action = 'upgrade' a held hero +1 level, or
               'new' = add a freshly drafted ball), draw_upgrade,
               hero_ability_blurb (player-facing text for each hero).

[Spatial queries] get_nearest_brick(_within), has_brick_within,
               get_bricks_within, get_random_brick_within — used by hero AI.

[Hero ability helpers]  Shared AoE/projectile primitives called by BallHero:
               fire_projectile_at_nearest, do_splash, do_chain_lightning,
               burn_area, slow_in_area, knockback_area.

[Powerups]     tick_powerup_pity + spawn_random_powerup (tier1/2 via chance/
               streak), tick_levelup_pity + spawn_levelup_powerup (the level
               orb on its own timer), apply_powerup(kind,x,y,color,amount)
               dispatch, tick_buffs + add_or_extend_buff (timed buffs),
               per-kind apply_* effects (apply_paddle_width_buff,
               apply_big_ball_buff, apply_fire_trail_buff, apply_freeze_wave,
               apply_water_wave, apply_multi_ball, apply_pierce_buff,
               apply_floor, apply_level_random), draw_buff_strip. Body-rebuild
               helpers rebuild_rect_body/rebuild_circle_body/resize_hero (used
               to resize paddle/balls for buffs).

--------------------------------------------------------------------------------
4. PLAYER FILES
--------------------------------------------------------------------------------
paddle.lua     : Paddle (kinematic, 36x4, restitution 1). Bullet-hell control:
                 A/D pan horizontally, W/S move in a vertical DODGE BAND
                 (80px up / 2px down from spawn y) to weave between bullets.
                 on_ball_bounce: edge-offset reflection (hit offset -> angle
                 in [-pi/2 ± pi/3]); ramps ball.speed_mult (×1.07, capped);
                 re-arms pierce buff; escalating pitch + sparks.

ball_hero.lua  : BallHero — the SNKRX heroes as bouncing balls.
                 - HERO_STATS table: ~58 characters, each with r, base_speed,
                   dmg, color, and a `behavior` (or `on_bounce`). Behavior
                   families (dispatched via a BEHAVIORS table):
                     shoot_arrow, shoot_knife, random_shot, multi_shot,
                     melee_splash, random_splash, heal, curse, dot_cloud,
                     bomb_drop, turret_drop, force_area, ally_buff_dmg,
                     ally_buff_aspd, pet_spawn, gambler_burst, time_dilation.
                   on_bounce exceptions (trigger per ball-bounce, not a timer):
                     wizard=chain_lightning, cryomancer=slow, pyromancer=burn,
                     cannoneer=big_splash, flagellant=flagellant_pulse.
                 - Continuous attackers fire when a brick is within `range`
                   (can_attack -> has_brick_within); projectile dmg ×3.5
                   (PROJECTILE_DMG_MULT). current_dmg = base × charge × ally buff.
                 - Ball physics: normalize_speed keeps constant speed; contact
                   damage on bounce (on_brick_hit); speed_mult ramps on paddle
                   bounce; pierce buff punches through bricks. Stuck/aim/launch
                   flow (start_stuck/launch_from_stuck) and returning flow
                   (start_return/update_return = magnet recall when it falls in
                   the pit). Fire trail, charge bonus, shaded tints for
                   same-color heroes.

--------------------------------------------------------------------------------
5. ENEMY FILES
--------------------------------------------------------------------------------
brick.lua      : Brick — one cell of a swarm (KINEMATIC; the swarm sets its
                 position each frame). VARIANTS table (stats tuned for this game,
                 ported from SNKRX): seeker, speed_booster, exploder, headbutter,
                 tank, shooter, spawner, swarmer, forcer, randomizer (melee/
                 utility) + sniper, spreader, spiraler, burster, arc_lobber
                 (ranged). hp scales ×(1+0.2*wave)×cell_count; xp ×cell_count.
                 Multi-cell tetris bodies (one fixture per cell, all routing to
                 one brick id). Behaviors via cast_* (speed_boost, headbutt,
                 shoot, sniper, spread, spiral, burst, arc_lob, spawn_critter,
                 force_push) and on-death (explode_on_death, swarmer_on_death).
                 RANGED_HOLD_FIRE_DIST=160 (ranged enemies stop firing point-
                 blank). on_ball_contact -> swarm knockback; take_damage/die ->
                 drop XP. BRICK_W/H=18x10, CELL_W/H=22x14 (grid must match
                 swarm planner).

swarm.lua      : Swarm — a 2D chunk of bricks moving as one springy unit
                 (Ball-x-Pit style). SHAPES catalogue (1x1 heavily weighted +
                 rectangles 1x1..3x3 + tetris T/L/J/S/Z, all fit in 3x3).
                 generate_cells() does row-major weighted occupancy placement
                 with `density` empties. Owns brick positions each frame; ball
                 hits apply a shared knockback impulse. Tracks breach (any
                 brick crossing the paddle line via brick:bottom_y()).

enemies.lua    : Non-brick mobiles + the boss.
                 - EnemyCritter: tiny circle, drifts down with wobble, dies in
                   1 hit, breaches paddle on contact; spawned by swarmer/spawner
                   bricks; drops 1 XP.
                 - EnemyProjectile: bullet-hell shot (slow, damages paddle, can
                   be shot down). Many per-kind draws (spike/dart/triangle/orb/
                   bolt/bomb/boss_orb/star/comet/diamond/armor); optional homing.
                 - AllyCritter: friendly pet (host/infestor/illusionist) that
                   flies UP and hits bricks.
                 - Boss (wave 10): 28r outer / 14r inner, kinematic, HP
                   2400×(1+0.2*wave). 3 PHASES tween color red->orange->purple
                   as HP drops, speeding the attack timer; phase 3 also drops
                   minions. Smooth parametric MOVEMENT modes. ATTACK POOL grows
                   per phase: p1 spiral/shotgun/snipe; p2 +ring/flower/wall;
                   p3 +spiral_double/homing (shotgun drops). All boss bullets
                   are flagged unbreakable and given a life floor (16s) so they
                   cross the tall arena. Supports slow/burn/curse like bricks.

6. PICKUPS
--------------------------------------------------------------------------------
xp_orb.lua     : XpOrb — dropped on brick death. Brief pop-out, gentle gravity,
                 then magnet-snaps to the paddle within range 130 (vampire-
                 survivors feel). value tints color (blue<green<yellow) and size.
                 Missed orbs fall through the open bottom and despawn.

powerup.lua    : Powerup — falling pickup, ONE per effect. (Heavily worked on
                 recently — see section 8.)
                 KINDS:
                   Tier 1 (instant catch + paddle magnet):
                     heal, wide_paddle, big_ball, fire_trail, freeze_wave.
                   Tier 2 (deflect-to-arm, then catch; "earn it"):
                     water_wave, multi_ball, pierce, floor,
                     level_random (solo; the level-up orb).
                 Helpers tier_1_kinds()/tier_2_kinds() exclude `solo` kinds.
                 All powerups are now FREE-ROTATING physics bodies that tumble
                 (set_fixed_rotation(false), random initial spin, angular
                 damping; :draw reads real body angle via get_angle). Catch is
                 a proximity box-check in :update (NOT a Box2D contact — see
                 collision matrix). apply_and_die -> arena:apply_powerup(kind,
                 x,y,color, level_amount).

8. RECENT WORK / CURRENT STATE  (powerup physics + level-orb mechanic)

Done this/last session, applied to BOTH builds' powerup.lua (those two files are
byte-identical) and threaded through ballpit.lua (both builds):

* Powerups tumble as real physics bodies (spin read from get_angle, not a clock).
* Tier-2 deflect imparts spin "english" with a floor so a dead-centre bounce
  never freezes the tumble (this is what makes tier-2 keep spinning).
* LEVEL ORB (level_random) is now a multi-bounce "earn it" pickup:
  - At spawn it pre-rolls level_amount = random:int(1,5) = the levels it grants
    AND the number of paddle bounces required (bounces_needed). Generic tier-2 =
    1 bounce. It only ARMS (becomes catchable) after deflect_count >=
    bounces_needed; the catch then grants that many levels (capped by eligible
    heroes in apply_level_random(amount)).
  - Remaining-bounce PIPS are drawn above the rune so the reward is previewed.
  - Gravity ramps each bounce: self.fall_gravity starts 60, +25 per bounce, used
    as the ballistic fall in :update — later bounces drop faster/harder.
  - Bounce launch speed = 240 (raised from 180) so it pops higher.
  - DESPAWN TIMER (self.life=16s, tagged 'despawn') is REFRESHED on every bounce
    (re-armed via self.t:after(...,'despawn')), so active play never times out
    mid-chain. It only despawns on 16s of inactivity or a whiff (falls past
    paddle). [Implemented to fix a "despawned before final bounce" bug.]
  - The rune body + up-arrow rotate with the physics body; the ripple rings stay
    radial. (NOTE: the rune was originally specced "static"; it now tumbles.)

* ballpit.lua: apply_powerup gained an `amount` param threaded to
  apply_level_random(amount); falls back to random:int(1,5) when nil (e.g. the
  admin terminal calls it without amount).

  bricks (brick.lua, both builds) is a "burn-line sweep": missing HP renders as

--------------------------------------------------------------------------------
9. COLLISION MATRIX  (set in BallPit:reset_run)
--------------------------------------------------------------------------------
World tags: paddle, ball, brick, wall, xp, projectile, powerup.
Collisions are ENABLED by default and selectively DISABLED:
  - ball:ball OFF.
  - xp collides with NOTHING (pure magnet logic in code).
  - projectile: collides with brick + wall(off) — i.e. only bricks; OFF vs
    paddle/ball/projectile/wall. (Hero projectiles damage bricks only.)
  - brick: OFF vs paddle/brick/wall (kinematic; adjacent bricks touch).
  - paddle:wall OFF.
  - POWERUP collides ONLY with walls (to stay in play). OFF vs paddle/ball/
    brick/projectile/xp/powerup. Catch + tier-2 deflect are driven by a
    proximity box-check in Powerup:update, NOT Box2D contacts — so spin/tumble
    never affects whether a powerup is caught.
  - Ball<->brick, ball<->wall, ball<->paddle stay ON (the core Breakout loop).

--------------------------------------------------------------------------------
11. GAME FLOW SUMMARY (one run)
--------------------------------------------------------------------------------
reset_run -> start_wave(1). Each wave: swarms spawn on a timer (wave_config),
drift down. Balls auto-attack bricks; killing bricks drops XP. XP -> level_up ->
3-card draft (add a new ball or +1 level an existing one). Powerups spawn on
pity timers; catch tier-1 instantly, deflect+catch tier-2, multi-bounce the
level orb for 1-5 hero levels. Combo points accrue from brick bounces and gate a
damage multiplier (D..FRENZY); missing a ball into the pit penalizes it. Bricks/
critters that reach the paddle line breach and cost player HP (5 total). Wave 9
ends -> hold for clear -> wave 10 BOSS (3 phases). Beat it -> waves 11+ (hardest
tier). player_hp <= 0 -> game over. (No win screen / no save persistence yet.)
